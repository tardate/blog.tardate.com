--- layout: post title: 'Released: Authlogic_RPX 1.1.1 gem now with identity mapping for Rails' date: '2010-01-20T13:48:00.005+08:00' author: Paul Gallagher tags: - Rails - Security - Ruby modified_time: '2010-01-20T19:42:03.411+08:00' thumbnail: http://1.bp.blogspot.com/_AZh6c1pzK1A/Sr45CYFXyaI/AAAAAAAABAI/ubfYuR3SRwk/s72-c/rpx-basic-highlevel2.png blogger_id: tag:blogger.com,1999:blog-6220586.post-8079008436478217939 blogger_orig_url: http://blog.tardate.com/2010/01/released-authlogicrpx-111-gem-now-with.html --- Authlogic_RPX is the gem I released some time back that lets you easily support multiple authentication schemes in Rails without really having to think about it (Facebook, twitter, OpenID, Google, Yahoo! etc etc etc).

With thanks for contributions from John and Damir, Authlogic_RPX 1.1.1 is now available and includes an internal identity mapping that lets you tie multiple authentication methods to a single user. There are also some other bug fixes and improvements, so it is recommended you update Authlogic_RPX even if you don't want to enable the identity mapping.

See the Authlogic_RPX repo on github for information about installation, usage, and don't forget the sample site (and sample code).
